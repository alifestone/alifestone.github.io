0. 快速開始（本機預覽）

所有內容都是靜態檔案，用 Astro 產生。改東西之前先把開發伺服器跑起來，邊改邊看：

npm run dev        # 開發模式，通常在 http://localhost:4321
npm run build      # 產生正式輸出到 dist/
npm run preview    # 預覽 build 後的結果
習慣： 每次改完都同時看 淺色 與 深色 兩種主題（右上角切換鈕），確認顏色與 Slimo 都正常。專案沒有 lint / test，肉眼驗證就是驗證。

1. 撰寫 Markdown 文章

一篇文章 = src/content/posts/ 底下一個 .md 檔。新增文章最簡單的做法：

1複製範本檔 src/content/posts/hello-world.md，改成新檔名，例如 my-first-ctf.md。檔名會變成網址：/posts/my-first-ctf/。
2改上方 --- 之間的 frontmatter（標題、日期、分類、標籤、摘要）。欄位說明見下一節。
3在 frontmatter 底下用 Markdown 寫正文。
4存檔 → 開發伺服器會自動重整。文章會自動出現在首頁與 /posts/ 列表，並依日期排序。
檔名建議： 用小寫英文與連字號（ppo-from-scratch.md），避免空白與中文，網址才乾淨。日期不用寫進檔名，排序看 frontmatter 的 date。

2. Frontmatter 欄位一覽

每篇文章最上方用兩行 --- 包起來的 YAML。這是唯一「規定格式」的地方——欄位定義在 src/content/config.ts，填錯 build 會報錯（這是好事，會擋下漏填）。

---
title: PPO 從零開始：一步步推導 clip objective
date: 2026-06-19
category: 強化學習
tags: [PPO, PyTorch, RL]
excerpt: 不依賴任何 RL 函式庫，從 policy gradient 推到 PPO-Clip，200 行驗證。
readMin: 8
draft: false
---
欄位必填說明
title是文章標題（字串）。
date是日期 YYYY-MM-DD。決定排序（新→舊）與顯示日期。
category是分類，只能是預先定義好的那幾個之一。要加新分類看 第 5 節。
tags否標籤陣列，如 [PPO, PyTorch]。沒填就是空陣列。詳見 第 4 節。
excerpt是摘要，顯示在文章卡片上。一兩句話。
cover否封面圖，相對路徑如 ./cover.png。
coverAlt否封面圖替代文字（無障礙用）。
readMin否閱讀分鐘數（數字），自己估。
draft否true = 草稿，不會出現在網站上；預設 false。
注意： date 不要加引號寫成字串，直接寫 2026-06-19；category 的值要跟定義完全一樣（含空白，如 資安 CTF 中間有一個半形空白）。

3. Markdown 語法速查

正文支援標準 Markdown。以下是這個部落格會用到、且套好樣式的元素（完整範例見 hello-world.md）。

標題、粗斜體、連結

## 這是 h2（會進目錄 TOC）
### 這是 h3（也會進目錄）

行內可以用 **粗體**、*斜體*、`inline code`，或放一個[連結](/about/)。
文章頁的目錄（TOC）會自動抓 ## 和 ###。用 #（h1）沒必要——標題已由 title 產生。

清單與引言

- 無序清單項目
- 第二項

1. 有序清單
2. 第二步

> 這是引言區塊（blockquote）。

程式碼區塊

三個反引號加語言名稱，會套用像素風深色外框：

```python
def clip_objective(ratio, adv, eps=0.2):
    return -torch.min(ratio*adv, ratio.clamp(1-eps,1+eps)*adv).mean()
```

表格

| 設定 | 成功率 |
|---|---|
| base | 12% |
| 微調 @ 20k | **74%** |

4. 新增／管理 tag

標籤是免登記的——不需要在任何清單裡預先定義。你只要在文章 frontmatter 的 tags 陣列寫下它，網站就會自動收集、去重、統計數量，並顯示在側欄與文章列表的篩選器。

tags: [LeRobot, SmolVLA, fine-tuning]
  • 新增 tag： 直接在某篇文章的 tags 加一個新字串即可。
  • 刪除 tag： 把它從所有文章的 tags 移掉，它就會自動消失（沒有文章用到的 tag 不會出現）。
  • 重新命名： 這是純字串比對，所以 PyTorch 和 pytorch 會被當成兩個不同 tag。想改名就到用到它的每篇文章一起改，保持大小寫一致。
背後邏輯在 src/lib/posts.ts 的 collectTags()：它掃過所有文章、依出現次數排序。你不用碰這支檔案，寫文章時管好 tags 陣列就好。

5. 新增分類

分類跟 tag 不同——它是受控清單，要新增（例如 開發、claude、certification）需要動幾支檔案。目前已有的分類：

機器人 資安 CTF 強化學習 日本語

下面以新增一個 開發 分類（chip class 取名 c-dev、顏色用天藍 #7EC8FF）為例，走一遍五個步驟：

1 登記到 schema — src/content/config.ts，把新名稱加進 category enum： 
category: z.enum(['機器人', '資安 CTF', '強化學習', '日本語', '開發']),
沒這一步，文章寫 category: 開發 會 build 失敗。
2 對應 chip class — src/lib/posts.ts，在 CAT_CLASS 加一行，並把它加進 collectCategories() 的 order 陣列（決定側欄顯示順序）： 
export const CAT_CLASS = {
  '機器人': 'c-robo',
  '資安 CTF': 'c-ctf',
  '強化學習': 'c-rl',
  '日本語': 'c-jp',
  '開發': 'c-dev',        // ← 新增
};
// ...
const order = ['機器人', '資安 CTF', '強化學習', '日本語', '開發']; // ← 也加進來
3 加顏色 — src/styles/global.css，找到這行加上新的 class： 
.c-robo{background:#7EE8C7}.c-ctf{background:#FF8A8A}.c-rl{background:#FFC145}.c-jp{background:#C4B5FD}.c-dev{background:#7EC8FF}
4 加進文章列表的篩選器 — src/pages/posts/index.astro，在分類陣列加一項： 
{ key: '開發', name: '開發', cls: 'c-dev' },
5 （選配）關於頁介紹 — 若想在 src/pages/about.astro 的「我在做什麼」區塊也介紹這個分類，比照現有格式加一項 { cat, cls, title, desc }。純介紹用，不影響文章功能。
小抄： 加分類 = 動 config.ts（登記）+ posts.ts（class 對應與順序）+ global.css（顏色）+ posts/index.astro（篩選）。前三支是必要的；漏了 config.ts 會 build 失敗，漏了 css 顏色 chip 會變透明。

之後只要在文章寫 category: 開發 就能用了。想加 claude、certification 等其他分類，同樣照這五步，換掉名稱、class 名與顏色即可。

6. 放圖片與封面

正文圖片

把圖片放在文章 .md 旁邊（同資料夾）或 src/assets/，用相對路徑引用，Astro build 時會自動最佳化：

![機械手臂抓透明杯](./grasp.png)

封面圖

在 frontmatter 指定 cover（會顯示在卡片／文章頂端），建議一起給 coverAlt：

cover: ./cover.png
coverAlt: 機械手臂抓取透明物件的示意圖
放在 public/ 的圖片則用絕對路徑 /xxx.png 引用、不會被最佳化，適合 favicon 這類固定資源。

7. 發佈上線

站點是 GitHub Pages user site（alifestone.github.io）。發佈流程：把改動 commit 後 push 到 main，GitHub Actions 會自動 build 並部署。

git add .
git commit -m "add: 新文章 xxx"
git push
Pages 的來源要設成 GitHub Actions（Settings → Pages → Source）。部署 workflow 在 .github/ 底下。push 後到 repo 的 Actions 分頁看綠燈就代表上線了。

8. 常見問題

文章寫好了卻沒出現？

  • 檢查 draft: 是不是還是 true。
  • 檔案要放在 src/content/posts/ 且副檔名是 .md。
  • frontmatter 有沒有兩行 --- 包好、必填欄位有沒有漏。

build 報錯 Invalid enum value？

category 填了沒登記的分類。回 第 5 節 先把它加進 config.ts 的 enum，或改成現有分類。

chip（分類標籤）變透明／沒顏色？

忘了在 global.css 加對應的 .c-xxx{background:…}。

想改 Slimo（吉祥物）或整體設計？

那屬於設計層，見 CLAUDE.md 的架構說明（Slimo 的 sprite 在 src/scripts/site.js，設計系統在 global.css）。日常寫文章不需要碰。
